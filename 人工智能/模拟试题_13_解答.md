## 三、计算题解答

### 1. A*算法求最短路径

**题目：** 求从S到G的最优路径，边代价和启发值如题

**解答：**
A*算法使用f(n) = g(n) + h(n)
初始：OPEN=[S(0+7=7)], CLOSED=[]
- 从S开始，f(S)=7
- 扩展S：S→A(5+5=10), S→B(2+4=6)，OPEN=[B(6), A(10)], CLOSED=[S]
- 选择B：f(B)=6，扩展B：B→C(2+3+2=7), B→D(2+7+1=10)，OPEN=[C(7), A(10), D(10)], CLOSED=[S, B]
- 选择C：f(C)=7，扩展C：C→G(2+3+2+0=7)，OPEN=[G(7), A(10), D(10)], CLOSED=[S, B, C]
- 选择G：f(G)=7，到达目标

**答案：** 最优路径S→B→C→G，总代价7

### 2. 神经网络计算

**题目：** 输入x=[1, 2]，Sigmoid激活函数（修正为Sigmoid以符合神经网络常用）

**解答：**
- 隐藏层输入：
  - h1_in = 1×0.1 + 2×0.3 + 0.1 = 0.8
  - h2_in = 1×0.2 + 2×0.4 + 0.2 = 1.2
- 隐藏层输出（Sigmoid）：
  - h1_out = 1/(1+e^(-0.8)) ≈ 0.690
  - h2_out = 1/(1+e^(-1.2)) ≈ 0.769
- 输出层计算：
  - output = 0.690×0.5 + 0.769×0.6 + 0.3 = 0.345 + 0.461 + 0.3 = 1.106

**答案：** 输出值约为1.106

## 四、证明题解答

**题目：** 从∀x(P(x) → Q(x))和¬Q(a)推导¬P(a)

**证明：**
使用归结原理：
1. 将前提转换为子句：
   - 子句1：¬P(x) ∨ Q(x) （∀x(P(x) → Q(x))）
   - 子句2：¬Q(a)
   - 子句3：P(a) （¬P(a)的否定）

2. 归结过程：
   - 步骤4：子句1（¬P(x) ∨ Q(x)）和子句2（¬Q(a)）归结（令x=a）→ ¬P(a)
   - 步骤5：步骤4结果（¬P(a)）和子句3（P(a)）归结 → 空子句

3. **结论：** 从假设P(a)推出矛盾，因此¬P(a)为真

## 六、简答题解答

**自然语言处理主要任务及方法：**

### 主要任务：
1. **分词：** 将连续文本分割为词汇单元
2. **词性标注：** 标注词汇的语法类别
3. **命名实体识别：** 识别人名、地名、机构名等实体
4. **句法分析：** 分析句子的语法结构
5. **语义分析：** 理解词语和句子的含义
6. **情感分析：** 判断文本的情感倾向
7. **机器翻译：** 自动翻译不同语言
8. **问答系统：** 回答用户自然语言问题

### 传统方法：
- **基于规则：** 语言学专家制定语法规则
- **统计方法：** 基于概率模型（n-gram、HMM、CRF）
- **机器学习：** SVM、决策树等传统算法
- **特点：** 依赖人工特征工程，可解释性强

### 深度学习方法：
- **词嵌入：** Word2Vec、GloVe学习分布式表示
- **神经网络：** RNN、LSTM、GRU处理序列数据
- **注意力机制：** 重点关注重要信息
- **Transformer：** 自注意力实现并行计算
- **预训练模型：** BERT、GPT等大规模预训练

### 传统 vs 深度学习比较：

| 方面 | 传统方法 | 深度学习方法 |
|------|----------|--------------|
| 特征工程 | 手动设计特征 | 自动特征学习 |
| 数据需求 | 小数据集有效 | 需要大量数据 |
| 计算复杂度 | 相对简单 | 计算量大 |
| 可解释性 | 较好 | 较差 |
| 性能 | 在特定任务上有效 | 在大数据上性能优越 |

### 中文NLP特殊挑战：
1. **分词问题：** 中文没有自然分隔符，歧义分词多
2. **语义理解：** 上下文依赖性强，多义词多
3. **语言现象：** 丰富的成语、俗语、省略现象
4. **数据资源：** 标注数据相对较少
5. **计算复杂度：** 词汇量大，计算复杂度高
6. **方言问题：** 普通话与方言混合使用

解决中文NLP挑战的策略包括：大规模预训练中文模型、专门的中文分词工具、构建中文知识库等。
