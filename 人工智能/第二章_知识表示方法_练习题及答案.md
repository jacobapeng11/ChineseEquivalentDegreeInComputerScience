## 三、计算/转换题答案

### 1. 谓词逻辑转换

**a) 所有人都会死**
答案：∀x(Human(x) → Mortal(x))
详解：对所有x，如果x是人，则x会死。

**b) 有些学生喜欢音乐**
答案：∃x(Student(x) ∧ Like(x, Music))
详解：存在x，x是学生且x喜欢音乐。

**c) 如果下雨，地面就会湿**
答案：∀x(Rain(x) → Wet(Ground))
详解：如果下雨，则地面湿（Ground为地面概念）。

### 2. 语义网络表示

答案：
```
      鸟
     /| |\
  ISA | | \
动物 羽毛 会飞 背有翅膀
  |    |   |    |
  企鹅 有  不会 有
        飞   翅
           膀
```

详解：
- Bird ISA Animal （鸟是一种动物）
- Bird Has-Property Feathers （鸟有羽毛）
- Bird Can Fly （鸟会飞）
- Bird Has-Part Wings （鸟有翅膀）
- Penguin ISA Bird （企鹅是一种鸟）
- Penguin Not Can Fly （企鹅不会飞）

## 四、简答题答案

### 1. 比较语义网络和框架表示法的异同点

**相同点：**
- 都属于结构化知识表示方法
- 都能表示概念及其关系
- 都支持继承机制
- 直观易懂

**不同点：**

| 方面 | 语义网络 | 框架 |
|------|----------|------|
| 结构形式 | 图状结构（节点-弧） | 层次结构（槽-值） |
| 表示重点 | 概念间的关系 | 对象的属性和特征 |
| 推理方式 | 图遍历、路径搜索 | 槽值匹配、继承 |
| 适用场景 | 概念关系、分类 | 实体建模、属性描述 |
| 表达能力 | 适合关系表达 | 适合特征描述 |

**详解：** 语义网络更适合表达概念之间的复杂关系，而框架更适合描述对象的属性和特征。

### 2. 解释产生式规则的推理过程，并举例说明

**推理过程：**
1. **匹配阶段：** 将工作内存中的事实与规则的前提条件进行匹配
2. **选择阶段：** 从匹配的规则中选择一条（冲突解决方案）
3. **执行阶段：** 执行选中规则的结论部分（添加/删除事实）
4. **重复：** 重复上述过程直到达到目标或无规则可匹配

**举例说明：**
```
规则库：
R1: IF 动物有毛发 THEN 动物是哺乳动物
R2: IF 动物是哺乳动物 AND 动物会下蛋 THEN 动物是鸭嘴兽
R3: IF 动物是哺乳动物 AND 动物有蹄子 THEN 动物是偶蹄动物

工作内存（初始事实）：
- 动物1有毛发
- 动物1会下蛋

推理过程：
1. R1匹配：动物1有毛发 → 动物1是哺乳动物（新事实添加）
2. R2匹配：动物1是哺乳动物 + 动物1会下蛋 → 动物1是鸭嘴兽
3. 无更多匹配规则，推理结束
```

**详解：** 产生式系统通过不断匹配和执行规则来实现推理。

## 五、应用题答案

### 1. 图书馆系统知识库设计

**设计：**

**1. 谓词逻辑表示：**
```
- Book(b1) ∧ Author(b1, "张三") ∧ Title(b1, "人工智能")
- Person(p1) ∧ Name(p1, "李四") ∧ Age(p1, 25)
- Borrow(p1, b1, Date1) ∧ ReturnDate(p1, b1, Date2)
```

**2. 语义网络表示：**
```
     书
     |
   ISA
     |
   图书《人工智能》
   /     |       \
作者   标题      类别
 |      |        |
张三   "AI"   计算机
```

**3. 框架表示：**
```
框架：书籍 (Book)
  槽：书号 (BookID)
  槽：书名 (Title)
  槽：作者 (Author)
  槽：出版日期 (PubDate)
  槽：是否可借 (Available)

框架：读者 (Reader)
  槽：读者号 (ReaderID)
  槽：姓名 (Name)
  槽：可借数量 (MaxBorrow)
```

**4. 产生式规则：**
```
R1: IF 读者没有逾期未还图书 AND 没达到最大借阅数 
    THEN 允许借阅
R2: IF 图书在馆藏中 AND 无人预约 
    THEN 图书可借
```

**各方法优缺点：**

**谓词逻辑：**
- 优点：逻辑严密，推理能力强
- 缺点：表达复杂关系困难，计算复杂度高

**语义网络：**
- 优点：直观，易于理解，支持继承
- 缺点：形式不够严格，推理机制有限

**框架：**
- 优点：结构清晰，便于表示默认值和约束
- 缺点：继承关系复杂时容易冲突

**产生式规则：**
- 优点：形式直观，易于理解和维护
- 缺点：规则间可能冲突，推理效率取决于冲突解决方案

**详解：** 在实际应用中，可结合使用多种方法，发挥各自优势，构建完整的知识系统。
