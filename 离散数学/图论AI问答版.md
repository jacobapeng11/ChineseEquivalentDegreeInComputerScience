## Q1：图 `G = <V, E, φ>` 里的“图”有三维概念吗，还是只是在平面上？

**A：** 图论中的图本质上是一个**抽象关系结构**，不是几何对象。

- `V` 是顶点集合
- `E` 是边集合
- `φ` 描述边和顶点之间的关联关系

这个定义里**没有坐标、没有长度、没有角度、没有二维或三维空间**。

我们平时在纸上或屏幕上画出来的图，只是把抽象图嵌入到平面（2D）或空间（3D）中的一种表示方式，叫 **embedding（嵌入）**。  
所以：

- 图本身：不带二维/三维概念
- 画出来的图：可以在二维、三维甚至更高维空间中表示

## Q2：无向图、有向图、简单图、多重图、混合图，这些分类是研究什么的？有什么意义？是谁命名的？

**A：** 这些分类不是为了“分类而分类”，而是为了**正确建模现实中的不同关系类型**。

### 为什么要分类？
现实世界中的关系不一样：

- 朋友关系：对称 → 无向图
- 引用关系：有方向 → 有向图
- 一个对象之间有多条不同连接：允许平行边 → 多重图
- 有的关系有方向，有的没方向：→ 混合图

### 分类的意义
不同的图类型允许研究不同问题：

- 无向图：连通性、最小生成树、社区结构
- 有向图：拓扑排序、PageRank、任务依赖、网络流
- 多重图：交通系统、多链路网络
- 混合图：复杂工作流、工业控制系统

### 谁命名的？
不是某一个人统一命名，而是图论、运筹学、计算机科学、工程等领域在长期发展过程中逐渐形成的术语体系。

## Q3：DAG 在 AI 计算图中的作用是什么？

**A：** 神经网络本质上可以表示成一个 **DAG（有向无环图）**。

- **有向**：表示计算有依赖方向
- **无环**：否则会出现循环依赖，无法计算

例如：

```text
z = (x + y) * w
```

可以拆成：

```text
m = x + y
z = m * w
```

这就是一个 DAG。

### DAG 在 AI 中的三个核心作用
1. **确定正向计算顺序**：通过拓扑排序执行 forward pass
2. **支持反向传播**：通过逆拓扑顺序传播梯度
3. **支持并行与内存优化**：框架可以根据图结构调度计算与显存

---

## Q4：什么叫“在 DAG 上做拓扑排序 + 反向图遍历”？请详细解释并举例。

**A：** 这是自动微分和反向传播的核心。

以：

```text
L = (a*b + c)^2
```

为例，拆成：

```text
m1 = a*b
m2 = m1 + c
L  = m2^2
```

### 1）正向传播 = 拓扑排序执行
合法计算顺序必须满足依赖关系，因此一种拓扑序列是：

```text
a, b, m1, c, m2, L
```

先算 `m1`，再算 `m2`，最后算 `L`。

### 2）反向传播 = 逆拓扑顺序遍历
从 `L` 出发反向算梯度：

- `dL/dL = 1`
- `dL/dm2 = 2*m2`
- `dL/dm1 = dL/dm2 * dm2/dm1`
- `dL/da = dL/dm1 * dm1/da`
- `dL/db = dL/dm1 * dm1/db`
- `dL/dc = dL/dm2 * dm2/dc`

如果 `a=2, b=3, c=4`：

- `m1 = 6`
- `m2 = 10`
- `L = 100`

梯度是：

- `∂L/∂a = 60`
- `∂L/∂b = 40`
- `∂L/∂c = 20`

## Q5：最后用 DAG 反向传播求出来的导数，和直接手算求导一样吗？

**A：** 一样，**完全一致**。

还是以：

```text
L = (ab + c)^2
```

为例，直接求导：

- `∂L/∂a = 2(ab+c)*b`
- `∂L/∂b = 2(ab+c)*a`
- `∂L/∂c = 2(ab+c)`

代入 `a=2,b=3,c=4`，得到：

- `∂L/∂a = 60`
- `∂L/∂b = 40`
- `∂L/∂c = 20`

与 DAG 反向传播结果完全相同。

### 区别在哪里？
区别不在数学结果，而在**计算方式**：

- 直接展开求导：会有很多重复子表达式
- DAG 反向传播：每个中间量只算一次，可复用

所以反向传播是链式法则的**程序化、动态图化执行**。

## Q6：公式里算出来 `L = 100`，这个数字代表什么意义？

**A：** 这个 `100` 表示的是 **Loss（损失函数值）**。

它的含义是：

> 当前参数下，模型“错得有多严重”

例如在回归里常用平方误差：

```text
L = (预测值 - 真实值)^2
```

如果误差是 10，那么 Loss 就是 100。

### Loss 和梯度的区别
- **Loss**：当前错得多大
- **Gradient**：应该往哪个方向调整参数，才能让 Loss 变小

所以训练过程就是：

```text
算 Loss → 算梯度 → 更新参数 → 再算 Loss
```

---

## Q7：为什么要分别对 `a、b、c` 求导？

**A：** 因为 `a、b、c` 是**不同的可调参数**，训练时需要知道：

- 改 `a` 会让 Loss 怎样变化？
- 改 `b` 会让 Loss 怎样变化？
- 改 `c` 会让 Loss 怎样变化？

数学上，这对应偏导数：

- `∂L/∂a`
- `∂L/∂b`
- `∂L/∂c`

它们共同组成梯度向量：

```text
∇L = [∂L/∂a, ∂L/∂b, ∂L/∂c]^T
```

这个向量告诉我们：

> 在参数空间里，Loss 上升最快的方向是什么

而梯度下降则沿着负梯度方向更新参数。

---

## Q8：AI 里的拓扑序列，和算法课上学的拓扑排序是同一个概念吗？

**A：** 是的，**完全是同一个概念**。

算法课里的拓扑排序：

> 对一个 DAG，排出一个线性顺序，使得若 `u → v`，则 `u` 一定在 `v` 前面。

AI 里的计算图也是 DAG：

- Forward pass：按拓扑序计算
- Backward pass：按逆拓扑序传播梯度

所以：

- 图论/算法中的拓扑排序 = AI 中的合法正向计算顺序
- 逆拓扑顺序 = 反向传播顺序

## Q9：握手定理是怎么发现的？为什么奇度顶点个数一定是偶数？

**A：** 握手定理来自一个非常自然的计数观察：

### 握手定理
```text
∑ deg(v) = 2|E|
```

原因是：每条边有两个端点，因此在统计所有顶点度数时，每条边都会被数两次。

### 奇度顶点为什么一定是偶数个？
因为总度数和 `2|E|` 一定是偶数。  
而若图中奇度顶点个数是奇数个，那么这些奇数度相加会得到奇数，与总和为偶数矛盾。

所以：

> 任意图中，奇度顶点个数必为偶数

## Q10：完全图、完全有向图、二部图、完全二部图，这些特殊图有什么意义？

**A：** 它们是现实系统里最常见、最基本的关系结构模型。

### 完全图 `K_n`
任意两个顶点之间都有边。表示“全连接系统”。

例子：
- 小团体中所有人互相认识
- Transformer 中自注意力可类比为 token 的完全连接图

### 完全有向图
任意两个顶点之间都有一对方向相反的有向边。表示“互相作用系统”。

### 二部图
顶点可分为两个集合 `V1`、`V2`，边只能跨集合连接。表示“两类实体之间的关系”。

例子：
- 用户—商品
- 老师—课程
- 工人—任务

### 完全二部图 `K_{m,n}`
`V1` 中每个顶点都与 `V2` 中每个顶点相连。是匹配问题、推荐系统、网络流建模的基础结构。

## Q11：邻接矩阵和关联矩阵到底是什么？请举例说明。

**A：** 它们都是把图转换成矩阵的方式，但表示的关系不同。

### 1）邻接矩阵（Adjacency Matrix）
表示“点—点是否相连”。

例如无向图：

```text
v1 — v2 — v3
```

邻接矩阵：

```text
[0 1 0]
[1 0 1]
[0 1 0]
```

### 2）关联矩阵（Incidence Matrix）
表示“点和边的关系”。

若边为：
- `e1 = v1-v2`
- `e2 = v2-v3`

无向图关联矩阵：

```text
[1 0]
[1 1]
[0 1]
```

对于有向图：

- 出发点记 `1`
- 到达点记 `-1`
- 无关记 `0`

## Q12：Transformer 的注意力矩阵是什么？为什么矩阵乘法像是在做逻辑推导？加减除又有什么用？

**A：** Transformer 的注意力矩阵来源于：

```text
Attention(Q,K,V) = softmax(QK^T / sqrt(d)) V
```

其中：
- `Q`：Query（我想找什么）
- `K`：Key（我有什么可被查）
- `V`：Value（真正要传递的信息）

### `QK^T` 在干什么？
它计算 token 与 token 的匹配分数：

```text
S_ij = q_i · k_j
```

含义是：

> token `i` 应该多大程度上关注 token `j`

这像是“模糊逻辑推理”或“语义相关性匹配”。

### 为什么矩阵乘法能表达推理？
矩阵乘法本质是**线性组合与特征重组**：

```text
y = Wx
```

表示把原有特征加权组合成新的抽象特征。深层堆叠后，就能形成复杂的语义推理能力。

### 加法、减法、除法的作用
- **加法**：融合信息
- **减法**：做差异、对比、去除某一属性
- **除法**（如 `/sqrt(d)`）：做归一化，避免点积值过大影响 softmax

## Q13：你之前给的那个“attention 矩阵”为什么不是 0/1？邻接矩阵不应该只有 0 和 1 吗？

**A：** 这里要区分两种“邻接矩阵”：

### 普通邻接矩阵
表示有没有边，所以是：

- 0：无边
- 1：有边

### 加权邻接矩阵
表示边的“强度”或“权重”，所以值可以是任意实数。

Transformer 的 attention 矩阵属于：

> **加权邻接矩阵**

因为它记录的是 token 之间的**语义相关强度**，不是简单的“连/不连”。

## Q14：那我怎么知道 `cat` 和谁重要？为什么 `(cat,cat)=5`、`(cat,on)=0`？

**A：** 一开始模型并不知道谁和谁重要，**这些权重都是训练学出来的**。

attention 分数由：

```text
S_ij = q_i · k_j = (W_Q x_i)^T (W_K x_j)
```

决定，而 `W_Q`、`W_K` 是模型参数，通过最小化 Loss 学得。

如果某种关联有助于降低预测误差，例如：

- 动词更应该关注主语
- 代词更应该关注其指代对象

那么反向传播就会把相应的 `q_i·k_j` 变大。

所以并不是“先验规则”写进去，而是：

> 哪些 token 之间的关联有助于预测，模型就会把它们学出来

---

## Q15：为什么 token 点积变大就说明是好事？Loss 高低怎么验证是否真的影响准确率？

**A：** 重点在于：

> 点积变大本身不是目标，**降低 Loss 才是目标**。

只有当 `q_i·k_j` 变大**有助于正确预测**时，梯度才会推动它变大；如果它让预测更差，Loss 会升高，训练会把它压回去。

### Loss 和准确率的关系
训练时常用的交叉熵 Loss 是一个可导目标，用来作为准确率的连续代理：

- Loss 降低 → 模型给正确答案分配的概率更高
- 验证集上一般会看到：Loss 下降，Accuracy 上升

所以“好不好”最终由任务目标决定，而不是由点积数值本身决定。

## Q16：邻接表是什么？

**A：** 邻接表是图的**链式存储结构**，本质是：

> 对每个顶点，记录它所有的邻居列表

例如图：

```text
v1 — v2 — v3
 |
 v4
```

邻接表可以写成：

```text
v1: v2 → v4
v2: v1 → v3
v3: v2
v4: v1
```

### 为什么需要邻接表？
因为邻接矩阵需要 `n^2` 空间，而很多图是稀疏图，边远少于 `n^2`。  
邻接表空间复杂度是：

```text
O(n + m)
```

更适合工程实现。

## Q17：途径、迹、道路、回路、闭迹、圈 这些概念难记，它们有什么意义？

**A：** 它们本质上是在区分：

> 走图时，是否允许重复使用“边”或“顶点”

### 一组直观理解
- **途径（walk）**：点可重复，边可重复
- **迹（trail）**：边不重复，点可以重复
- **道路/路径（path）**：顶点不重复（边自然也不重复）
- **回路（closed walk）**：起点终点相同
- **闭迹（closed trail）**：回到起点且边不重复
- **圈（cycle）**：回到起点且除首尾外顶点不重复

### 为什么有意义？
不同实际问题对“重复资源”的限制不同：
- 清扫街道：希望每条边只走一次 → 欧拉问题
- 访问城市：每个顶点只访问一次 → 哈密顿问题
- DAG 检测环：关心是否存在圈 → cycle detection

## Q18：连通图、连通分量、点连通度、边连通度是什么意思？`κ(G) ≤ λ(G) ≤ δ(G)` 有什么意义？

**A：** 这些概念都在研究：

> 图在被破坏时，有多“抗断裂”

### 基本概念
- **连通图**：任意两顶点间都有通路
- **连通分量**：图中极大的连通子图
- **点连通度 `κ(G)`**：最少删多少个顶点，图会断开或变平凡图
- **边连通度 `λ(G)`**：最少删多少条边，图会断开
- **最小度 `δ(G)`**：所有顶点度数中的最小值

### 定理
```text
κ(G) ≤ λ(G) ≤ δ(G)
```

### 含义
- 删点往往比删边更容易造成网络瘫痪
- 最弱顶点的度数给出了边连通度的一个上界

这是评估网络鲁棒性的基本理论。

## Q19：证明：在 `n` 阶简单图中，若每个顶点度数都 `≥ (n-1)/2`，则图连通。能不用反证法，用归纳思想说明吗？

**A：** 可以，用“逐步扩展可达顶点集合”的构造思想来理解。

### 思路
从任意顶点 `v0` 出发，记：

```text
S = 从 v0 可达的所有顶点集合
```

如果 `S` 不是全图，设 `|S| = k < n`。

由于 `S` 外的点和 `S` 内无边，否则就能扩展进来，因此：

- `S` 内任意点的度最多是 `k-1`
- 但题设要求度至少是 `(n-1)/2`
- 所以 `k ≥ (n+1)/2`

另一方面，对 `V\S` 中的点，同理可得：

- `n-k ≥ (n+1)/2`
- 即 `k ≤ (n-1)/2`

矛盾。

因此 `S` 不可能停在真子集，必须扩展到整个图，故图连通。

## Q20：欧拉图、半欧拉图是什么？为什么判定条件和奇偶度有关？

**A：** 欧拉问题关注的是：

> 能不能把图中所有边恰好走一遍

### 定义
- **欧拉迹**：经过图中所有边且每条边只走一次的迹
- **欧拉回路**：经过所有边且每条边只走一次，并回到起点的回路
- **欧拉图**：存在欧拉回路的图
- **半欧拉图**：存在欧拉迹但不存在欧拉回路的图

### 为什么和度数有关？
在欧拉回路中，经过每个中间顶点时，都必须“进一次、出一次”，所以每个顶点贡献的边数必须成对出现，即度数为偶数。

### 无向图判定
- 欧拉图 ⇔ 连通且所有顶点度为偶数
- 半欧拉图 ⇔ 连通且恰有两个奇度顶点

### 有向图判定
欧拉回路存在当且仅当每个顶点：

```text
入度 = 出度
```

## Q21：Königsberg 七桥问题为什么走不通？

**A：** 把陆地看作顶点，把桥看作边，得到一个图。  
这个图有 **4 个奇度顶点**。

根据欧拉图定理：

- 若存在欧拉回路，奇度顶点数必须为 0
- 若存在欧拉迹，奇度顶点数必须为 2

而它有 4 个奇度顶点，所以：

> 无论怎么走，都不可能把 7 座桥不重复地走完并回到起点

这就是图论的历史起点之一。

---

## Q22：哈密顿图是什么？为什么它比欧拉图难得多？

**A：** 哈密顿问题关注的是：

> 能不能把图中所有顶点恰好访问一遍

### 定义
- **哈密顿路**：经过所有顶点的道路
- **哈密顿回路**：经过所有顶点并回到起点的圈
- **哈密顿图**：存在哈密顿回路的图
- **半哈密顿图**：存在哈密顿路但没有哈密顿回路

### 为什么比欧拉图难？
因为欧拉图判定依赖顶点的局部度数条件，而哈密顿性是一个**全局性质**，不能仅凭局部信息决定。

### 经典充分条件
- **Dirac 定理**：若 `deg(v) ≥ n/2`，则图是哈密顿图
- **Ore 定理**：若任意不相邻顶点 `u,v` 都满足 `deg(u)+deg(v) ≥ n`，则图是哈密顿图

但目前没有像欧拉图那样简单的充要条件。

## Q23：平面图的欧拉公式 `n - ε + r = 2` 里，为什么有多个“面”？平面图不是只有一个面吗？

**A：** 这里的“面（face）”不是“整张纸”，而是：

> 图的边把平面分割出来的区域

例如一个三角形图：

- 内部三角区域：一个面
- 外部无限区域：也是一个面

所以一个三角形图有：

```text
n = 3, ε = 3, r = 2
```

满足：

```text
3 - 3 + 2 = 2
```

### 关键点
平面图的面包括：
- 所有内部封闭区域
- 外部的无限区域

## Q24：为什么对简单平面图有 `ε ≤ 3n - 6`？

**A：** 由欧拉公式和“每个面至少有 3 条边”推出。

### 推导
平面简单图中：
- 每个面至少由 3 条边围成
- 每条边恰好属于两个面的边界

所以：

```text
3r ≤ 2ε
```

由欧拉公式：

```text
n - ε + r = 2
```

解出：

```text
ε ≤ 3n - 6
```

### 意义
它给出了平面图边数的上界。若某图边太多，必定不是平面图。

例如：
- `K5`：`n=5, ε=10`
- 但 `3n-6 = 9`
- 所以 `K5` 不是平面图

---

## Q25：`3r ≤ 2ε` 里，为什么是 `3×面数`？边数又为什么乘 `2`？

**A：** 这里数的是：

> 所有面的边界总长度

### 为什么有 `3r`？
因为简单平面图中每个面至少由 3 条边围成，所以 `r` 个面的边界总长度至少是：

```text
3r
```

### 为什么有 `2ε`？
因为每条边都位于两个面的公共边界上：

- 一边是它左边的面
- 一边是它右边的面

所以当我们把所有面的边界长度加起来时，每条边会被数两次，于是总长度恰好是：

```text
2ε
```

因此有：

```text
3r ≤ 2ε
```

## Q26：Kuratowski 定理是什么意思？为什么偏偏是 `K5` 和 `K3,3`？

**A：** Kuratowski 定理说：

> 一个图是平面图，当且仅当它不含与 `K5` 或 `K3,3` 同胚的子图

### “同胚”是什么意思？
允许在边上插入若干个度为 2 的中间点，也就是把一条边“拉长”，但不改变整体连接结构。

### 为什么是 `K5` 和 `K3,3`？
因为它们是最基本的两类**不可平面结构障碍**：

- `K5`：五点完全图
- `K3,3`：三三完全二部图

只要图内部“藏着”它们（哪怕经过边的细分变形），这个图就一定画不成平面图。

## Q27：极大平面图是什么？“我管它是不是极大平面图”——研究它的意义是什么？

**A：** 极大平面图不是为了起个特殊名字，而是为了研究：

> 在保持平面性的前提下，图能稠密到什么程度？

### 定义
若在图中任意两个不相邻顶点之间加一条边，都会破坏平面性，则称该图为 **极大平面图**。

### 核心性质
对于 `n ≥ 3` 的极大平面图：

- `ε = 3n - 6`
- 每个面都是三角形

### 为什么每个面都是三角形？
如果某个面不是三角形，比如是四边形，就还可以加一条对角线而不破坏平面性，这与“极大”矛盾。

### 意义
极大平面图表示：

> **平面图在“不交叉”约束下所能达到的最大连接密度**

这在三角剖分、网格划分、芯片布线等问题中非常重要。

# 最后总总结

## 图论部分
- 图本质是关系结构，不自带几何维度
- 图分类是为了准确建模不同关系类型
- 路径、迹、圈区分的是是否允许重复顶点/边
- 握手定理来自双计数
- 连通度刻画网络抗破坏能力
- 欧拉问题关注“边遍历”
- 哈密顿问题关注“顶点遍历”
- 平面图研究的是“能否无交叉嵌入平面”

## AI 部分
- 神经网络计算图本质是 DAG
- Forward = 拓扑排序，Backward = 逆拓扑遍历
- 反向传播是链式法则在图上的执行
- Attention 矩阵是 token 完全图的加权邻接矩阵
- 点积不是目标，Loss 最小化才是目标

## 平面图部分
- 欧拉公式：`n - ε + r = 2`
- 平面图边数上界：`ε ≤ 3n - 6`
- 不可平面的根本障碍：`K5` 与 `K3,3`
- 极大平面图达到平面图允许的最大连接密度
