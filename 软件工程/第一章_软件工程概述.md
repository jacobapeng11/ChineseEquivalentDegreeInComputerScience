# 第一章：软件工程概念与软件工程基本要素

## 概述

软件工程是一门研究用工程化方法构建和维护有效的、实用的和高质量的软件的学科。它将系统化的、规范化的、可量化的方法应用于软件的开发、运行和维护过程，即将工程化方法应用于软件。本章介绍软件工程的基本概念、基本要素以及软件危机等核心内容。

### 软件的定义与特点

**软件的定义：**
软件是计算机系统中与硬件相互依存的另一部分，它包括程序、数据及相关文档的完整集合。其中，程序是按事先设计的功能和性能要求执行的指令序列；数据结构是使程序能够正确地处理信息的数据结构；文档是与程序开发、维护和使用有关的图文材料。

**软件的特点：**
1. **无形性：** 软件是无形的，不可见的逻辑实体
2. **复杂性：** 软件的逻辑复杂性远超其他工程产品
3. **可变性：** 软件需要不断修改以适应变化的需求
4. **可复制性：** 软件可以无损耗地复制
5. **依赖性：** 软件依赖于硬件和运行环境

### 软件危机

**软件危机的定义：**
软件危机是指在计算机软件的开发和维护过程中所遇到的一系列严重问题，包括软件开发进度难以预测、开发成本难以控制、软件质量不可靠、软件维护困难等。

**软件危机的表现：**
1. 对软件开发成本和进度的估计很不准确
2. 用户对已完成的软件系统不满意的现象时常发生
3. 软件产品的质量往往靠不住
4. 软件常常是不可维护的
5. 软件通常缺少适当的文档资料
6. 软件开发的生产率提高很缓慢

**软件危机产生的原因：**
1. 软件本身的复杂性
2. 用户需求的不明确
3. 开发人员对需求理解的偏差
4. 缺乏正确的理论指导
5. 软件开发的管理困难

### 软件工程的定义与目标

**软件工程的定义：**
软件工程是应用计算机科学、数学及管理科学等原理，以工程化的原则和方法来解决软件问题的工程。其目的是建立高质量的软件，并提高软件开发的效率和可维护性。

**软件工程的目标：**
1. 降低软件开发成本
2. 提高软件质量
3. 提高软件开发效率
4. 确保软件可维护性
5. 提高软件的可靠性

### 软件工程的基本要素

软件工程包含三个基本要素：方法、工具和过程。

1. **方法：** 软件工程方法为软件开发提供"如何做"的技术。它涉及需求分析、软件设计、编码、测试和维护等一系列活动的方法。

2. **工具：** 软件工具为软件工程方法的实施提供自动化的支撑环境。工具包括需求分析工具、设计工具、编程工具、测试工具、维护工具等。

3. **过程：** 软件工程过程是指将软件工程的方法和工具综合起来以达到合理、及时地开发软件的目的。过程定义了方法使用的顺序、要求交付的文档资料、对质量和评审设定的管理点等。

### 软件工程的7个基本原理

1. **分阶段的生命周期计划：** 严格定义软件生命周期各个阶段的任务和目标
2. **坚持阶段评审：** 在每个阶段结束前进行严格的评审
3. **实行严格的产品控制：** 采用配置管理等工具控制变更
4. **采用现代程序设计技术：** 采用先进的开发方法和工具
5. **明确的责任制：** 建立明确的组织和责任制度
6. **用产品实测标准来衡量进度：** 采用定量化的进度度量方法
7. **坚持不断地改进软件工程实践：** 持续改进软件开发过程

## 软件生命周期过程

### 软件生命周期的概念

软件生命周期是指软件产品从提出、实现、使用维护到停止使用退役的整个过程。根据软件工程考纲要求，软件生存周期过程包括基本过程、支持过程和组织过程。

### 软件生存周期的三大过程

1. **基本过程：** 与软件生产直接相关的活动集，包括：
   - 获取过程：需方获取软件系统或软件产品的活动
   - 供应过程：供方提供软件系统或软件产品的活动
   - 开发过程：将需求转化为软件产品的活动
   - 运行过程：软件产品在运行环境中的活动
   - 维护过程：软件产品交付后的修改和演化活动

2. **支持过程：** 支持软件产品生产活动的技术活动集，包括：
   - 文档过程：文档的编制和管理
   - 配置管理过程：标识、控制和报告软件项的变化
   - 质量保证过程：确保软件产品满足需求
   - 验证过程：确认软件项是否适当地满足了特定的要求
   - 确认过程：确保软件项满足其特定的用途
   - 联合评审过程：对软件、软件描述及相关文档进行评价
   - 审计过程：独立评价软件产品或过程对特定要求的遵循程度
   - 问题解决过程：问题的控制和解决

3. **组织过程：** 与软件生产组织直接相关的活动集，包括：
   - 管理过程：项目管理、配置管理等
   - 基础设施建设过程：建立和维护软件工程环境
   - 改进过程：改进软件开发过程
   - 培训过程：为相关人员提供培训

## 软件生存周期模型

### 主要的软件开发模型

根据考纲要求，软件生存周期模型包括以下几种：

1. **瀑布模型：** 传统的线性开发模型
   - 优点：结构化、文档驱动、适用于需求明确的项目
   - 缺点：缺乏灵活性、反馈滞后、无法适应需求变更

2. **增量模型：** 分批交付软件产品
   - 优点：降低风险、客户可及早使用软件、容易维护
   - 缺点：软件体系结构要求高、集成困难

3. **演化模型：** 通过原型逐步演进的方式
   - 优点：适应需求变化、用户参与度高
   - 缺点：管理复杂、文档可能不完整

4. **螺旋模型：** 结合瀑布模型和原型模型，以风险为驱动
   - 优点：风险驱动、适合大型复杂项目
   - 缺点：复杂度高、成本高、依赖风险评估

5. **喷泉模型：** 体现软件开发过程的迭代性和无间隙性
   - 优点：支持软件的重用、迭代开发
   - 缺点：管理复杂、需要严格的控制

## 软件项目生存周期过程的规划与监控

### 项目生存周期过程定义

软件项目生存周期过程的定义是根据项目特点、组织环境和需求，确定适合的软件生存周期模型和过程活动。

### 项目生存周期过程规划

1. **确定生存周期模型：** 根据项目特点选择合适的模型
2. **定义活动和任务：** 明确各阶段的具体活动
3. **分配资源：** 为各项活动分配人员、时间、资金等资源
4. **确定里程碑：** 设置关键的检查点和交付物

### 项目生存周期过程监控

1. **进度监控：** 跟踪项目实际进度与计划的偏差
2. **质量监控：** 确保各阶段交付物满足质量要求
3. **成本监控：** 控制项目成本在预算范围内
4. **风险监控：** 识别、评估和应对项目风险

## 能力成熟度模型(CMM)

### CMM的基本概念

能力成熟度模型(CMM)是卡内基梅隆大学软件工程研究所(SEI)提出的软件过程评价与改进框架，用于指导软件组织改进其软件过程能力。

### CMM的五个等级

1. **初始级(Initial)：** 软件过程的特征是无序的，有时甚至是混乱的。成功主要依赖个人能力。
2. **可重复级(Repeatable)：** 建立了基本的项目管理过程，能对软件项目进行跟踪和控制。
3. **已定义级(Defined)：** 管理活动和工程活动的软件过程均已文档化、标准化，并集成为组织的标准软件过程。
4. **已管理级(Managed)：** 对软件过程和产品质量都有定量的理解和控制。
5. **优化级(Optimizing)：** 过程的量化反馈和新技术的创新使软件过程不断改进。

### 各等级的关键过程域和基本特征

每个等级都有相应关键过程域(KPA)来描述要达到该等级必须解决的软件过程问题，以及基本实践(BP)来指导如何解决这些过程问题。

## 本章练习题答案解析

### 选择题答案

1. **答案：D**
   - 解析：软件工程的基本要素是方法、工具和过程，不包括代码。

2. **答案：B**

3. **答案：C**

4. **答案：B**

5. **答案：C**
   - 解析：开发过程属于基本过程，不是支持过程。

### 简答题答案

6. **答案：**
   软件工程的基本要素包括：
   1) 方法：为软件开发提供"如何做"的技术，包括需求分析、设计、编码、测试等方法
   2) 工具：为软件工程方法的实施提供自动化的支撑环境，如分析工具、设计工具、测试工具等
   3) 过程：将软件工程的方法和工具综合起来以达到合理、及时地开发软件的目的，定义了方法使用的顺序、要求交付的文档等

7. **答案：**
   CMM的五个等级：
   1) 初始级：过程无序，成功依赖个人能力
   2) 可重复级：建立基本项目管理过程，能跟踪控制项目
   3) 已定义级：软件过程文档化、标准化，成为组织标准
   4) 已管理级：对过程和产品质量有定量的理解和控制
   5) 优化级：通过量化反馈和创新不断改进软件过程

8. **答案：**
   瀑布模型优点：结构化、文档驱动、适用于需求明确的项目；缺点：缺乏灵活性、反馈滞后、无法适应需求变更。

   螺旋模型优点：风险驱动、适合大型复杂项目、结合了其他模型的优点；缺点：复杂度高、成本高、依赖风险评估。
