# 软件工程模拟试题 05 - 详细解答

## 一、选择题解答（每题2分，共20分）

1. **答案：B**
   **解析：** 甘特图主要用于显示项目进度，以条形图的形式展示任务的开始时间、结束时间和持续时间。

2. **答案：D**
   **解析：** 功能性可移植性不属于功能性范畴，可移植性是独立的质量特性。功能性范畴包括功能适合性、准确性、完整性、互操作性和依从性。

3. **答案：D**
   **解析：** 依赖倒置原则（Dependency Inversion Principle）的定义是：高层模块不应该依赖低层模块，两者都应该依赖抽象；抽象不应该依赖细节，细节应该依赖抽象。

4. **答案：B**
   **解析：** 敏捷开发中，Sprint长度通常为2-4周，这既短到能够保持适应性，又长到能够完成有意义的工作。

5. **答案：D**
   **解析：** 单元测试属于开发阶段的测试，不属于系统测试范畴。系统测试包括功能测试、性能测试、安全测试等。

6. **答案：A**
   **解析：** CASE全称为Computer Aided Software Engineering，即计算机辅助软件工程。

7. **答案：C**
   **解析：** 单一职责原则（Single Responsibility Principle）指出，一个类应该只有一个引起变化的原因。

8. **答案：D**
   **解析：** 软件维护的四个主要类别是改正性维护、适应性维护、完善性维护和预防性维护，没有重构性维护这一类别。

9. **答案：B**
   **解析：** 基线是指已经通过正式评审和批准的规格说明或产品，它可以作为进一步开发的基础。

10. **答案：B**
    **解析：** 软件危机的根本原因是软件规模和复杂性不断增加，而软件开发技术没有相应地提高。

## 二、填空题解答（每空2分，共20分）

11. **答案：** 可移植性
    **解析：** 软件工程的目标涉及ISO/IEC 25010质量模型中的多个特性。

12. **答案：** 螺旋
    **解析：** 螺旋模型将风险分析纳入开发过程，适合大型复杂系统的开发。

13. **答案：** 需求验证
    **解析：** 需求工程包括需求获取、分析、规格说明、验证和管理等活动。

14. **答案：** 路径
    **解析：** 路径覆盖是最强的白盒测试覆盖标准，要求覆盖程序中的所有路径。

15. **答案：** 观察者
    **解析：** 观察者模式定义对象间的一对多依赖关系。

16. **答案：** 项目
    **解析：** 软件项目管理的4P是指People（人员）、Product（产品）、Process（过程）和Project（项目）。

17. **答案：** 逻辑
    **解析：** McCabe圈复杂度测量程序的逻辑复杂度。

18. **答案：** 项目工作产品
    **解析：** 软件配置项是指配置管理中需要单独管理和控制的项目工作产品。

19. **答案：** 产品待办列表(Product Backlog)
    **解析：** 在Scrum框架中，产品待办列表是按优先级排序的需求列表。

20. **答案：** SQA
    **解析：** Software Quality Assurance的缩写。

## 三、简答题解答（每题10分，共30分）

21. **答案：**
    (1) 风险分析过程包括风险识别、风险估计、风险评价和风险应对（2分）
    (2) 风险识别：识别可能影响项目的潜在风险（2分）
    (3) 风险估计：评估风险的概率和影响程度（2分）
    (4) 风险评价：确定风险的重要性和优先级（2分）
    (5) 风险应对：制定风险缓解策略（1分）
    (6) 常用方法：头脑风暴、德尔菲法、SWOT分析等（1分）

22. **答案：**
    (1) 重要性：体系结构是系统的骨架，影响系统的质量属性（2分）
    (2) 影响系统的性能、安全性、可维护性等特性（2分）
    (3) 体系结构风格包括：分层结构（1分）
    (4) 客户端-服务器结构（1分）
    (5) 面向服务结构（1分）
    (6) 事件驱动结构（1分）
    (7) 微服务结构（1分）
    (8) 不同风格适应不同的应用场景（1分）

23. **答案：**
    (1) 质量与成本的关系：提高质量通常增加初期成本，但会降低后期维护成本（3分）
    (2) 质量缺陷修复成本随开发阶段推进呈指数增长（2分）
    (3) 平衡方法：在早期投入更多质量保证活动（2分）
    (4) 采用合适的设计、评审、测试方法（2分）
    (5) 建立质量标准和度量体系（1分）

## 四、综合分析题解答（每题15分，共30分）

24. **答案：**
    (1) 建议采用安全开发生命周期（SDL）过程（2分）
    (2) 需求分析阶段：详细分析安全性、可靠性、性能需求（2分）
    (3) 重点考虑安全威胁建模（1分）
    (4) 体系结构设计阶段：设计安全的系统架构（2分）
    (5) 重点考虑安全防护措施（1分）
    (6) 详细设计阶段：设计安全的组件和接口（1分）
    (7) 编码阶段：遵循安全编码规范（1分）
    (8) 测试阶段：进行安全测试、渗透测试（2分）
    (9) 部署阶段：建立安全的运行环境（1分）
    (10) 维护阶段：持续监控安全状况（2分）

25. **答案：**
    (1) 项目管理工具主要类型：
    - 项目计划工具（如Microsoft Project）（1分）
    - 配置管理工具（如Git、SVN）（1分）
    - 问题跟踪工具（如JIRA）（1分）
    - 协作工具（如Teams、Slack）（1分）
    
    (2) 选择工具的考虑因素：
    - 项目规模和团队大小（1分）
    - 项目特性（1分）
    - 成本预算（1分）
    - 易用性和学习曲线（1分）
    - 集成能力（1分）
    
    (3) 工具使用的影响：
    - 提高效率（1分）
    - 改善沟通（1分）
    - 增强透明度（1分）
    - 但需注意工具不应成为负担（1分）
    - 合理使用工具可以显著提升项目成功率（1分）
