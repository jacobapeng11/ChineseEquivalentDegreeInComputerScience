## 课程概览

软件工程是一门研究用工程化方法构建和维护有效的、实用的和高质量的软件的学科。本课程涵盖了软件工程的核心理论和实践方法，包括软件工程基础、需求工程、设计、测试、维护、项目管理、质量保证以及新兴技术等内容，为理解和应用软件工程技术打下坚实基础。

### 课程知识结构
- **基础理论：** 软件工程概述、软件生命周期
- **开发过程：** 需求分析、系统设计、编码实现、测试验证
- **支持活动：** 软件维护、配置管理、项目管理
- **质量保证：** 质量模型、质量度量、质量保证
- **发展趋势：** 敏捷开发、DevOps、云原生、AI应用

## 各章核心知识点

### 第一章：软件工程概述
- **软件的定义与特点：** 无形性、复杂性、可变性、可复制性、依赖性
- **软件危机：** 表现、原因及解决方案
- **软件工程的7个基本原理：** 分阶段计划、阶段评审、产品控制、现代技术、责任制、标准度量、持续改进
- **软件生命周期模型：** 瀑布模型、原型模型、增量模型、螺旋模型、敏捷模型
- **软件工程方法学：** 传统方法学、面向对象方法学、面向服务方法学

### 第二章：需求工程
- **需求的定义与分类：** 功能需求、非功能需求、约束
- **需求工程过程：** 获取、分析、规约、验证、管理
- **需求发现技术：** 访谈、问卷、观察、会议、原型
- **需求分析与建模：** 结构化分析、面向对象分析、原型方法
- **软件需求规格说明书：** 内容、特性(SRS的8个特性)
- **需求验证与管理：** 评审、变更控制、跟踪

### 第三章：软件设计
- **软件设计的目标与原则：** 正确性、可理解性、可维护性等；模块化、抽象、信息隐藏等
- **软件体系结构：** 分层、客户端-服务器、SOA、微服务、事件驱动
- **模块化设计：** 内聚(功能内聚→顺序内聚→通信内聚)、耦合(非直接→数据→控制→公共→内容)
- **面向对象设计：** 六大设计原则、设计模式
- **详细设计：** 数据结构、算法、接口设计、表示方法
- **用户界面设计：** 原则、过程、可用性

### 第四章：软件测试
- **软件测试的定义与目标：** 发现错误、验证功能、评估质量、建立信心
- **软件测试的基本原则：** 显示缺陷、穷尽不可能、尽早介入、缺陷集群、杀虫剂悖论
- **测试分类：** 按阶段(单元、集成、系统、验收)、按技术(黑盒、白盒、灰盒)
- **测试用例设计：** 等价类划分、边界值分析、因果图、判定表、正交表
- **白盒测试：** 语句覆盖、判定覆盖、条件覆盖、路径覆盖
- **测试阶段：** 单元测试、集成测试(自顶向下、自底向上、混合、大爆炸)、系统测试、验收测试
- **自动化测试：** 优势、工具、适用场景

### 第五章：软件维护与演化
- **维护分类：** 改正性、适应性、完善性、预防性维护
- **维护过程：** 申请、分析、设计、实现、测试、文档更新、部署
- **可维护性：** 易分析性、易修改性、稳定性、易测试性
- **维护副作用：** 代码、数据、文档副作用
- **软件再工程：** 逆向工程、重构、正向工程、体系结构重建
- **配置管理：** 配置项、版本控制、变更控制

### 第六章：软件项目管理
- **项目计划：** 内容、WBS构建原则
- **进度管理：** 甘特图、网络图、关键路径法、PERT
- **成本管理：** 估算方法、COCOMO模型
- **质量管理：** 规划、保证、控制、全面质量管理
- **风险管理：** 识别、分析、应对、监控；技术风险、管理风险
- **团队管理：** 民主型、主程序员型、层次型团队
- **CMM模型：** 五个成熟度级别

### 第七章：软件质量与标准
- **质量模型：** ISO/IEC 25010(八大特性)、McCall模型(三大维度)
- **质量度量：** 缺陷密度、故障率、MTTF、MTTR、圈复杂度
- **质量保证：** 标准制定、过程审计、产品审计、质量改进
- **软件评审：** 走查、检查、同行评审、代码评审
- **软件标准：** ISO/IEC 12207、CMMI、IEEE标准
- **质量文化：** 领导承诺、全员参与、持续改进

### 第八章：软件工程新技术与发展趋势
- **敏捷开发：** 敏捷宣言、12原则、Scrum、XP、看板
- **DevOps：** 文化、自动化、度量、共享；CI/CD、IaC
- **云原生技术：** 微服务、容器、Kubernetes
- **AI在软件工程中的应用：** 代码生成、审查、缺陷检测、测试生成
- **发展趋势：** 智能化、低代码、边缘计算

## 重要概念对比

### 传统方法 vs 敏捷方法
| 特征 | 传统方法 | 敏捷方法 |
|------|----------|----------|
| 开发模式 | 瀑布模型 | 迭代增量 |
| 文档 | 详尽文档 | 工作软件优先 |
| 变更处理 | 难以适应 | 拥抱变化 |
| 客户参与 | 早期和晚期 | 持续参与 |

### 黑盒测试 vs 白盒测试
| 特征 | 黑盒测试 | 白盒测试 |
|------|----------|----------|
| 测试依据 | 功能需求 | 程序结构 |
| 关注点 | 输入输出行为 | 代码逻辑路径 |
| 测试者 | 通常非开发人员 | 开发人员 |
| 发现问题 | 功能错误 | 逻辑错误 |
| 用例设计 | 等价类、边界值 | 语句、路径覆盖 |

### 五种内聚类型对比（从高到低）
1. **功能内聚：** 模块内所有元素共同完成一个功能
2. **顺序内聚：** 模块内元素顺序执行
3. **通信内聚：** 模块内元素操作同一数据
4. **过程内聚：** 模块内元素按特定顺序执行
5. **时间内聚：** 模块内元素在同一时间执行

### 五种耦合类型对比（从低到高）
1. **非直接耦合：** 模块间无直接联系
2. **数据耦合：** 模块间通过参数交换数据
3. **特征耦合：** 传递数据结构
4. **控制耦合：** 模块间传递控制信息
5. **公共耦合：** 多个模块访问公共数据

## 实践应用指南

### 需求工程实践
1. **需求获取时**：使用多种技术，确保用户参与
2. **需求分析时**：建立分析模型，验证一致性
3. **需求规约时**：使用标准模板，确保SRS特性
4. **需求管理时**：建立基线，控制变更

### 设计实践
1. **体系结构设计**：根据质量属性选择合适的架构风格
2. **模块化设计**：遵循高内聚低耦合原则
3. **面向对象设计**：应用设计模式，遵循SOLID原则
4. **用户界面设计**：以用户为中心，进行可用性测试

### 测试实践
1. **测试计划**：早期制定，覆盖所有测试级别
2. **用例设计**：结合黑盒和白盒技术
3. **测试执行**：自动化回归测试，手动探索测试
4. **缺陷管理**：及时记录，合理分配优先级

### 项目管理实践
1. **项目计划**：制定详细WBS，明确里程碑
2. **风险管理**：定期风险评估，制定应对策略
3. **质量保证**：建立质量标准，持续改进
4. **团队协作**：建立有效沟通机制

## 考试重点提醒

### 重点概念
1. **软件工程7个基本原理** - 可能出简答题
2. **需求工程过程** - 需理解和应用
3. **设计原则** - 高内聚低耦合是核心
4. **测试用例设计技术** - 重点掌握黑盒和白盒方法
5. **CMM五个级别** - 必考内容
6. **ISO/IEC 25010八大质量特性** - 重要知识点

### 常考题型
1. **概念题**：软件工程基本概念、定义
2. **比较题**：不同方法、模型、技术的比较
3. **应用题**：在特定场景下的应用分析
4. **计算题**：圈复杂度、成本估算等
5. **论述题**：综合分析和论述

### 复习建议
1. **系统性复习**：按章节顺序全面复习
2. **重点突破**：熟练掌握核心概念
3. **练习应用**：多做练习题，特别是计算题
4. **关注趋势**：了解最新发展如敏捷、DevOps等
5. **模拟考试**：限时完成模拟题，提高应试能力
