# 软件工程模拟试题 03 - 详细解答

## 一、选择题解答（每题2分，共20分）

1. **答案：B**
   **解析：** CMMI全称为Capability Maturity Model Integration（能力成熟度模型集成），是用于指导软件组织改进其软件过程能力的框架。

2. **答案：A**
   **解析：** 在软件需求规格说明中，最重要的是正确性，即准确反映用户的真实需求。其他特性虽然也很重要，但正确性是根本。

3. **答案：A**
   **解析：** 等价类划分法通过将输入域划分为若干等价类，从中选择代表性的测试数据，从而确定测试的输入数据。

4. **答案：C**
   **解析：** 面向服务结构（SOA）强调功能模块（服务）间的松散耦合，各个服务可以独立部署和演化。

5. **答案：A**
   **解析：** 开闭原则（Open-Closed Principle）是面向对象设计的基本原则之一，指对扩展开放，对修改关闭。

6. **答案：D**
   **解析：** 基线在配置管理中起着重要作用，包括标识关键点、控制变更、管理版本等多种功能。

7. **答案：B**
   **解析：** WBS全称为Work Breakdown Structure，即工作分解结构，是项目管理中的重要工具。

8. **答案：B**
   **解析：** 回归测试的主要目的是验证软件修改后没有引入新的错误或导致其他功能失效。

9. **答案：B**
   **解析：** 可靠性子特性包括成熟性、容错性、易恢复性等，易用性是独立的质量特性。

10. **答案：C**
    **解析：** 在Scrum框架中，产品负责人（Product Owner）负责维护和优先级排序产品待办事项列表（Product Backlog）。

## 二、填空题解答（每空2分，共20分）

11. **答案：** 过程
    **解析：** 软件工程的三大要素是方法、工具和过程。

12. **答案：** 数据流
    **解析：** 数据流图的四个基本元素是外部实体、处理过程、数据流和数据存储。

13. **答案：** 独立路径
    **解析：** 路径覆盖要求程序中的每条独立路径至少被执行一次。

14. **答案：** 行为型
    **解析：** 设计模式分为创建型、结构型和行为型三类。

15. **答案：** 预防性
    **解析：** 软件维护的四种类型包括预防性维护。

16. **答案：** 优化
    **解析：** CMMI模型的五个等级中最高级是优化级。

17. **答案：** 2p（或2，其中p为连通分量数）
    **解析：** McCabe圈复杂度公式为V(G) = e - n + 2p。

18. **答案：** 最长
    **解析：** 关键路径是项目中最长的路径，决定了项目的最短完成时间。

19. **答案：** Computer-Aided Software Engineering
    **解析：** CASE是Computer-Aided Software Engineering的缩写。

20. **答案：** 参者（Actor）
    **解析：** 用例图描述参与者与系统功能（用例）的交互关系。

## 三、简答题解答（每题10分，共30分）

21. **答案：**
    软件工程的基本原理包括：
    (1) 按分阶段的生命周期计划严格管理（2分）
    (2) 坚持阶段评审（2分）
    (3) 实行严格的产品控制（2分）
    (4) 采用现代程序设计技术（1分）
    (5) 明确的责任制（1分）
    (6) 用产品实测标准来衡量进度（1分）
    (7) 坚持不断地改进软件工程实践（1分）
    
    重要性：这些原理是软件工程实践的基础，能有效避免软件危机，提高软件开发效率和质量。

## 四、综合分析题解答（每题15分，共30分）

24. **答案：**
    (1) 面向对象分析基本步骤（5分）：
    - 识别对象/类（1分）
    - 识别属性（1分）
    - 识别服务/方法（1分）
    - 识别关系（继承、聚合、关联）（1分）
    - 构建分析模型（用例图、类图、序列图等）（1分）
    
    (2) 设计模式的应用（5分）：
    设计模式提供已验证的解决方案，如单例模式保证类只有一个实例，工厂模式封装对象创建过程。在电商系统中，可以使用观察者模式实现订单状态更新通知机制，使用策略模式根据不同促销规则计算价格。
    
    (3) 面向对象测试（5分）：
    - 类测试：针对单个类的所有方法（2分）
    - 方法测试：测试类中单个方法的功能（1分）
    - 关系测试：测试继承、聚合等关系（2分）
    使用框架如Junit进行单元测试。
