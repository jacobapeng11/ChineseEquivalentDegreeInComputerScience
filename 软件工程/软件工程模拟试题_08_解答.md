6. **答案：C**
   **解析：** 用例建模是需求建模技术，不是需求发现技术。需求发现技术包括用户访谈、原型法、观察法等。

## 二、填空题解答（每空2分，共20分）

11. **答案：** 圈复杂度
    **解析：** 圈复杂度是用来衡量程序复杂度的指标，反映代码的逻辑复杂性。

12. **答案：** 状态
    **解析：** 配置项在不同时间点通过评审后的稳定状态。

13. **答案：** Sprint评审（Sprint Review）
    **解析：** Sprint Review是Sprint结束后展示已完成工作的会议。

14. **答案：** 策略
    **解析：** 策略模式将算法封装起来，使它们可以相互替换。

15. **答案：** 需求跟踪矩阵
    **解析：** 需求跟踪矩阵是需求可追溯性的关键工具。

16. **答案：** 判定（分支）
    **解析：** 判定覆盖要求程序中每个判定的每个分支至少执行一次。

17. **答案：** 三层
    **解析：** 三层架构将系统划分为表示层、业务逻辑层和数据访问层。

18. **答案：** Infrastructure as Code (IaC)
    **解析：** 基础设施即代码是将基础设施当作代码进行管理。

19. **答案：** 燃尽（Burndown）
    **解析：** 燃尽图显示项目进度随时间变化的趋势。

20. **答案：** 预防性
    **解析：** 预防性维护是为了提高软件性能而进行的活动。

## 三、简答题解答（每题10分，共30分）

21. **答案：**
    (1) 概念：关注点分离是将程序分解为不同部分，每个部分处理一个独立的关注点，从而降低系统的复杂性（3分）
    (2) 应用：
    - 在体系结构层面：分层架构分离表示、业务、数据关注点（2分）
    - 在设计层面：MVC模式分离模型、视图、控制器（2分）
    - 在代码层面：高内聚低耦合原则（2分）
    (3) 好处：提高可维护性、可扩展性、可重用性（1分）

22. **答案：**
    (1) 飏险管理过程：
    - 飍险识别：识别潜在的项目风险（2分）
    - 飍险分析：评估风险概率和影响（2分）
    - 飍险评估：确定风险优先级（1分）
    - 飍险应对：制定应对策略（2分）
    - 飍险监控：跟踪风险状态（1分）
    (2) 方法：
    - 定性方法：风险矩阵、头脑风暴（1分）
    - 定量方法：蒙特卡洛模拟、决策树（1分）

23. **答案：**
    (1) 概念：测试左移是指在软件开发生命周期早期引入测试活动，提前发现和修复缺陷（3分）
    (2) 重要性：
    - 降低修复成本：早期发现缺陷成本更低（2分）
    - 提高质量：在开发过程中持续验证（2分）
    - 快速反馈：及时发现和解决问题（2分）
    - 集成测试前置：减少后期集成问题（1分）
