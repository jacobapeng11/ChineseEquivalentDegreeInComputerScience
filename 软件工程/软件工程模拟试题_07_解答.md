## 二、填空题解答（每空2分，共20分）

11. **答案：** 故事点（Story Points）
    **解析：** 故事点是敏捷开发中评估用户故事相对复杂度的单位。

12. **答案：** 側容性（Robustness）
    **解析：** 容错性测试验证软件在异常条件下的行为。

13. **答案：** 类（Class）
    **解析：** 类图描述系统的静态结构，包括类、属性、方法和类之间的关系。

14. **答案：** 代码规模（Size）
    **解析：** COCOMO是Constructive Cost Model，基于代码规模进行成本估算。

15. **答案：** 工厂方法（Factory Method）
    **解析：** 工厂方法模式将对象的创建延迟到子类。

16. **答案：** 进度（Time）
    **解析：** 项目管理的铁三角是范围、成本和时间。

17. **答案：** 易操作性（Operability）
    **解析：** 易用性的子特性包括易理解性、易学性、易操作性、吸引性。

18. **答案：** 基线（Baseline）
    **解析：** 基线是配置管理中标识软件开发过程中相对稳定点的版本。

19. **答案：** 主干（或Trunk）
    **解析：** 持续集成是频繁地将代码集成到主干版本中。

20. **答案：** 回顾（Retrospective）
    **解析：** Retrospective会议用于回顾团队的过程、人、关系、交互等。
