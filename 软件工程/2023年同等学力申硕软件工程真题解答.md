# 2023年同等学力申硕软件工程真题解答

> 本文件只保留 2023 年试卷中的软件工程部分，并整理成便于复习的详细解析版。

## 一、单项选择题（每题 1 分，共 5 分）

### 1. 软件开发中的需求活动，其主要任务是什么？

**答案：D 定义需求并建立系统模型**

**解析：**

- 需求分析阶段的核心任务是弄清楚“系统应该做什么”。
- 给出模块结构属于设计阶段。
- 定义模块算法属于详细设计阶段。
- 程序编码属于实现阶段。

### 2. 软件总体设计结束后得到什么？

**答案：B 初始（或初步）的软件结构图**

**解析：**

- 总体设计又叫概要设计，重点是模块划分、层次结构与接口设计。
- 这一阶段的典型产物是初始（或初步）软件结构图，后续还可继续细化和优化。

### 3. 面向对象分析方法与结构化分析方法的重要区别之一是什么？

**答案：B 一种以问题域成分为基点构造模块，而另一种不是**

**解析：**

- 面向对象分析从对象、类、属性、行为以及对象之间的关系出发。
- 结构化分析则更强调功能分解和数据流。

### 4. 确定测试计划是在什么阶段制定的？

**答案：A 需求分析**

**解析：**

- 软件测试不是等编码完成后才开始考虑。
- 测试计划应尽早制定，通常在需求分析阶段就要明确测试目标、策略和资源安排。

### 5. 按不同人员的工作内容来分类，软件基本过程包括什么？

**答案：C 获取过程和供应过程等**

**解析：**

- 软件过程按主体和活动划分时，基本过程包括获取、供应、开发、运行和维护。
- 这道题主要考查软件生命周期过程分类。

## 二、判断题（每题 1 分，共 5 分）

### 1. 结构化方法总的指导原则是自顶向下、逐步求精。

**答案：√**

**解析：**

- 结构化设计的核心思想就是先总体后局部、先抽象后具体。

### 2. 一个模块直接修改另一个模块的数据，这种耦合称为数据耦合。

**答案：×**

**解析：**

- 这属于内容耦合，不是数据耦合。
- 数据耦合是模块之间只通过参数传递基本数据，耦合程度相对较低。

### 3. 测试用例只需要选取合理的输入数据。

**答案：×**

**解析：**

- 测试用例不仅要有输入，还应明确预期输出、执行条件和判定标准。

### 4. 边界值分析法是一种白盒测试方法。

**答案：×**

**解析：**

- 边界值分析法属于黑盒测试技术。
- 它根据输入输出边界来设计测试用例，而不依赖程序内部结构。

### 5. 关联是建立类之间关系的一种手段。

**答案：√**

**解析：**

- 在 UML 和面向对象建模中，关联就是类与类之间的一类基本关系。

## 三、简答题（每题 4 分，共 12 分）

### 1. 给出模块耦合的定义，并举例说明 3 种模块耦合。

**参考答案：**

耦合是对不同模块之间相互依赖程度的度量。

可举例如下：

- 内容耦合：一个模块直接访问或修改另一个模块内部数据。
- 公共耦合：多个模块共同使用同一个全局数据项。
- 控制耦合：一个模块通过控制参数影响另一个模块的执行逻辑。
- 标记耦合：模块之间通过数据结构传递信息。
- 数据耦合：模块之间只通过参数传递基本数据。

**答题技巧：**

- 先给出“耦合是依赖程度度量”的定义。
- 再写 3 种耦合的名称和一句话解释，就足够拿分。

### 2. 将伪代码转换成 N-S 图和 PAD 图。

**参考答案：**

题目的关键不是图画得多漂亮，而是结构必须正确。程序控制流应写成：

1. 顺序执行 $S1$、$S2$。
2. 判断 $a>5$，若真执行 $S3$，否则执行 $S4$。
3. 执行赋值 $n=1$。
4. 进入循环：当 $n\le 5$ 时重复执行 $S5$ 和 $n=n+1$。
5. 循环结束后判断 $b>5$，若真执行 $S6$，否则执行 $S7$。

**评分关键：**

- 图中要体现两个条件分支。
- 要体现一个先判定后执行的循环结构。
- 顺序位置不能错，尤其是 $n=1$ 在循环之前。

**N-S 图：**

```text
┌──────────────────────────────────────────────┐
│ 顺序执行：S1 → S2                           │
├──────────────────────────────────────────────┤
│ 判断 a > 5                                  │
│ ├─ 是：执行 S3                              │
│ └─ 否：执行 S4                              │
├──────────────────────────────────────────────┤
│ 赋值 n = 1                                  │
├──────────────────────────────────────────────┤
│ while n ≤ 5                                 │
│ ├─ 执行 S5                                  │
│ └─ n = n + 1                                │
├──────────────────────────────────────────────┤
│ 判断 b > 5                                  │
│ ├─ 是：执行 S6                              │
│ └─ 否：执行 S7                              │
└──────────────────────────────────────────────┘
```

**PAD 图：**

```text
S1
S2
├─ a > 5 ?
│  ├─ 是 → S3
│  └─ 否 → S4
├─ n = 1
├─ while n ≤ 5
│  ├─ S5
│  └─ n = n + 1
└─ b > 5 ?
	├─ 是 → S6
	└─ 否 → S7
```

### 3. 给出 CMM 的 5 级模型，并举出某一级的关键过程域。

**参考答案：**

CMM 五级模型从低到高为：

- 初始级
- 可重复级
- 已定义级
- 已管理级
- 持续优化级

例如，可重复级的关键过程域包括：

- 需求管理
- 软件项目规划
- 软件项目跟踪与监督
- 软件质量保证
- 软件配置管理

也可以写其他等级，只要级别和关键过程域对应正确即可。

## 四、建模题（共 8 分）

题目背景：简化图书管理系统。

### 1. 顶层 DFD（系统环境图）应该如何画？

**核心对象：**

- 外部实体：库房管理员、图书管理员、图书馆主任、借阅人。
- 处理中心：图书管理系统。

**核心数据流：**

- 借阅人向系统提交借书、还书或图书丢失信息。
- 系统向借阅人返回无书通知或罚款单。
- 图书馆主任向系统发出查看库存请求。
- 系统向库房管理员发送购书通知。
- 图书管理员和库房管理员分别与系统交换入库、目录维护等信息。

### 2. 数据字典应包含哪些关键条目？

**参考答案：**

- 入库单 = 分类目录号 + 流水号 + 书名 + 作者 + 价格 + 购书日期
- 借还书的规定 = 借书信息 或 还书信息
- 借书信息 = 书名 + 作者 + 借书日期
- 还书信息 = 书名 + 还书日期
- 无书通知单 = 借阅人图书证号 + 无书信息
- 罚款单 = 借阅人图书证号 + 罚款金额
- 查看规定 = 分类目录号
- 月末库存量 = {分类目录号 + 书名 + 库存量}
- 购书通知 = 分类目录号 + 书名 + 购置量
- 图书丢失单 = 图书流水号 + 相关丢失信息

### 3. 次序图如何描述？

**可行的标准答案：**

可以选择“图书馆主任根据库存情况通知库房管理员购书”这一交互：

1. 图书馆主任向“图书库存信息文件”发起库存查询。
2. 系统返回某一分类目录号对应的月末库存情况。
3. 若库存量低于下限，图书馆主任向库房管理员发送购书通知。

**阅卷重点：**

- 交互对象是否合理。
- 消息方向是否正确。
- 是否体现了“先查询库存，再决定是否购书”的先后关系。

2023 年软件工程部分的命题重心主要有三类：

- 基础概念题：需求分析、总体设计、软件过程、测试方法。
- 简答题：耦合、流程图转换、CMM。
- 建模题：DFD、数据字典、次序图。
