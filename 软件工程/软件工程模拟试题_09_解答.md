# 软件工程模拟试题 09 - 详细解答

## 一、选择题解答（每题2分，共20分）

1. **答案：B**
   **解析：** 软件架构是系统的组织结构以及构成系统的各个组件之间的关系，是系统设计的高层次结构。

2. **答案：B**
   **解析：** 版本控制的主要目的是跟踪和管理软件的不同版本，记录修改历史。

3. **答案：C**
   **解析：** 观察者模式属于行为型模式，用于定义对象间的一对多依赖关系。

4. **答案：D**
   **解析：** Scrum框架包括三个角色：产品负责人、Scrum Master、开发团队，没有项目经理角色。

5. **答案：C**
   **解析：** 在V模型中，需求分析对应系统测试，概要设计对应集成测试。

6. **答案：C**
   **解析：** 一致性是数据一致性概念，不属于ISO/IEC 25010质量模型的特性，该模型包含功能性、性能效率、兼容性、易用性、可靠性、安全性、可维护性、可移植性。

7. **答案：C**
   **解析：** WBS是Work Breakdown Structure的缩写，即工作分解结构。

8. **答案：D**
   **解析：** 敏捷宣言强调"响应变化高于遵循计划"，而不是"严格计划高于响应变化"。

9. **答案：B**
   **解析：** 圈复杂度的计算公式为V(G) = e - n + 2p，其中e为边数，n为节点数，p为连通分量数。

10. **答案：B**
    **解析：** CMMI模型的五个等级从低到高：初始级(1)、已管理级(2)、已定义级(3)、量化管理级(4)、优化级(5)。

## 二、填空题解答（每空2分，共20分）

11. **答案：** 依赖倒置
    **解析：** 依赖倒置原则指出抽象不应该依赖细节，细节应该依赖抽象。

12. **答案：** 软件维护
    **解析：** 软件维护是指软件交付使用后为了改正错误或满足新需求而修改软件的过程。

13. **答案：** 速率（Velocity）
    **解析：** 速率是衡量Scrum团队在Sprint中完成工作量的指标。

14. **答案：** 基线
    **解析：** 基线是标识软件开发过程中相对稳定的特定时刻的配置项集合。

15. **答案：** 单元
    **解析：** 单元测试是验证单个模块或组件功能的测试。

16. **答案：** 安全性
    **解析：** 在ISO/IEC 25010模型中，安全性特性关注软件保护信息和数据的能力。

17. **答案：** 甘特（Gantt）
    **解析：** 甘特图用于显示项目活动的时间安排和依赖关系。

18. **答案：** 软件测试
    **解析：** 这是软件测试的定义。

19. **答案：** 持续集成（CI）
    **解析：** 持续集成是自动化构建、测试和部署代码的实践。

20. **答案：** SRS
    **解析：** Software Requirements Specification的缩写。

## 三、简答题解答（每题10分，共30分）

21. **答案：**
    (1) 架构风格概念：架构风格定义了系统组织方式的粗粒度结构，包括系统构件、连接件及约束（2分）
    (2) 常见架构风格：
    - 分层架构：将系统划分为多个层，上层依赖下层（2分）
    - 客户端-服务器：分离用户界面和服务实现（1分）
    - 面向服务：通过服务接口实现松耦合（1分）
    - 事件驱动：基于事件和消息传递（1分）
    - 微服务：将应用分解为小型独立服务（1分）
    - 管道-过滤器：数据流经一系列处理组件（1分）
    - 委托-控制：组件间通过委托关系协作（1分）

22. **答案：**
    (1) 黑盒测试：
    - 关注功能需求（1分）
    - 基于规格说明（1分）
    - 无需了解内部结构（1分）
    - 用户视角（1分）
    
    (2) 白盒测试：
    - 关注内部结构（1分）
    - 基于代码逻辑（1分）
    - 需了解内部实现（1分）
    - 开发者视角（1分）
    
    (3) 联系：都用于发现软件缺陷，提高质量（2分）

23. **答案：**
    (1) 核心价值观（4分）：
    - 个体和互动高于流程和工具
    - 可工作的软件高于详尽的文档
    - 客户合作高于合同谈判
    - 响应变化高于遵循计划
    
    (2) 12条原则（任选6项，每项1分）：
    - 通过早期和持续交付有价值的软件来满足客户
    - 欢迎需求变化，即使在开发后期
    - 经常交付工作软件，间隔时间短
    - 业务人员和开发人员必须每天一起工作
    - 围绕有动力的个人构建项目
    - 面对面交谈是最有效的沟通方式
    - 可工作的软件是进展的主要尺度
    - 敏捷过程提倡可持续的开发
    - 持续关注技术卓越和良好设计
    - 简单是必不可少的
    - 最佳架构、需求和设计源于自组织团队
    - 定期反思和调整

## 四、综合分析题解答（每题15分，共30分）

    - 实施自动化测试（1分）
    - 强化代码审查（1分）
    - 建立持续集成/持续部署流程（1分）
