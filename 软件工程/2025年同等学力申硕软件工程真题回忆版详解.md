# 2025年同等学力申硕软件工程真题回忆版详解

## 一、选择题详解

### 1. 螺旋模型

**标准答案要点：**

- 螺旋模型融合了瀑布模型的系统性和原型法的迭代思想。
- 它以风险分析为核心，适合大型、复杂、高风险项目。

### 2. 敏捷开发

**标准答案要点：**

- 强调快速迭代、持续交付、客户协作和响应变化。
- 不是否定文档，而是强调“可工作的软件”优先于“过度文档”。

### 3. 软件生命周期模型辨析

**标准答案要点：**

- 瀑布模型：阶段清晰，顺序推进。
- 增量模型：分批交付。
- 迭代模型：反复演进。
- 螺旋模型：迭代加风险分析。

## 二、填空题详解

### 1. 软件工程三要素

**答案：过程、方法、工具**

**解析：**

- 过程说明怎么组织开发。
- 方法说明如何分析、设计、实现和测试。
- 工具说明用什么来支持这些活动。

### 2. 需求分析的基本任务

**答案要点：**

- 获取需求
- 分析需求
- 规格说明
- 验证需求

**解析：**

- 本质上就是把用户的自然语言需求转换成准确、完整、无歧义的需求规格说明。

### 3. 软件设计过程

**答案要点：**

- 总体设计（概要设计）
- 详细设计

**解析：**

- 总体设计解决“系统如何分解”。
- 详细设计解决“模块内部如何实现”。

## 三、简答题详解

### 1. 瀑布模型的主要特点

**参考答案：**

- 开发阶段划分清晰，按顺序推进。
- 每一阶段都有相对明确的输入、输出和评审点。
- 文档驱动特征明显。
- 适合需求明确、变动较少的项目。
- 缺点是反馈周期长，难以适应频繁变化。

### 2. 黑盒测试与白盒测试的区别

**参考答案：**

**黑盒测试：**

- 根据需求规格说明设计测试用例。
- 关注输入与输出。
- 不关心程序内部结构。

**白盒测试：**

- 根据程序内部结构和逻辑设计测试用例。
- 关注语句覆盖、分支覆盖、路径覆盖等。

**对比记忆：**

- 黑盒看功能。
- 白盒看结构。

### 3. 面向对象设计的主要优点

**参考答案：**

- 提高可重用性：类和继承机制便于复用。
- 提高可维护性：封装后修改影响范围更清晰。
- 提高可扩展性：便于增加新类和新行为。
- 有利于建模现实世界：对象、属性、行为更贴近问题域。

### 4. 基础概念类简答题的通用答法

如果原卷第 4 题考的是基础概念，建议使用“定义 + 特点 + 作用”三段式作答：

1. 先给出规范定义。
2. 再写 2 到 3 条核心特点。
3. 最后补一句在软件开发中的作用。

## 四、论述题详解

### 1. 软件需求分析的重要性与主要原则

**参考答案：**

需求分析是软件开发的基础，直接影响后续设计、编码、测试和维护质量。若需求阶段出现偏差，后续代价会成倍放大。

需求分析应遵循的主要原则包括：

- 完整性：需求覆盖系统应具备的主要功能与约束。
- 一致性：各项需求之间不应互相矛盾。
- 可行性：需求在技术、成本和进度上可实现。
- 无歧义性：描述应明确，避免多种理解。
- 可验证性：需求应可通过测试、评审等方式验证。
- 可追踪性：需求应能追踪到设计、实现和测试。

### 2. 常见软件架构风格及其特点和应用场景

**参考答案：**

常见软件架构风格包括：

- 分层架构：结构清晰，适合管理信息系统和企业应用。
- 客户机 / 服务器架构：便于资源集中管理，适合传统管理系统。
- MVC 架构：分离模型、视图和控制器，适合 Web 应用。
- 微服务架构：服务解耦、便于独立部署，适合大型互联网系统。
- 管道-过滤器架构：数据流清晰，适合编译器、数据处理系统。

答题时建议按“定义 + 特点 + 典型场景”展开，而不是只列名称。

### 3. 现代软件开发生命周期中的项目风险管理

**参考答案：**

软件项目风险管理通常包括以下步骤：

1. 风险识别：识别需求变更、人员流失、技术不成熟、工期压缩等风险。
2. 风险分析：评估风险发生概率和影响程度。
3. 风险排序：确定优先级，把资源优先投入高风险项。
4. 风险应对：制定规避、缓解、转移或接受策略。
5. 风险监控：在项目执行过程中持续跟踪并更新风险状态。

在现代开发中，可通过迭代交付、持续集成、代码评审、自动化测试、需求澄清会议等方式降低风险。
